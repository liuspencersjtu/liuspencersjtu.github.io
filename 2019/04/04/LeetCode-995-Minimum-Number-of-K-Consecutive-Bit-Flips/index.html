In an array A containing only 0s and 1s, a K-bit flip consists of choosing a (contiguous) subarray of length K and simultaneously changing every 0 in the subarray to 1, and every 1 in the subarray to 0.
Return the minimum number of K-bit flips required so that there is no 0 in the array. If it is not possible, return -1.

Example 1:

Input: A = [0,1,0], K = 1
Output: 2
Explanation: Flip A[0], then flip A[2].

Example 2:

Input: A = [1,1,0], K = 2
Output: -1
Explanation: No matter how we flip subarrays of size 2, we can’t make the array become [1,1,1].

Example 3:

Input: A = [0,0,0,1,0,1,1,0], K = 3
Output: 3
Explanation:
Flip A[0],A[1],A[2]: A becomes [1,1,1,1,0,1,1,0]
Flip A[4],A[5],A[6]: A becomes [1,1,1,1,1,0,0,0]
Flip A[5],A[6],A[7]: A becomes [1,1,1,1,1,1,1,1]

一道leetcode上hard tag的题目， 难度准确来说大概算是medium和hard中间吧。从解题思路来看，首先要看出来这是一个贪心选择的问题：当我们从左到右进行翻转操作的时候，如果当前位置是0，则就一定要立刻翻转，因为在此之后，后面的翻转操作就没有办法影响到这个bit了。如果我们当前bit为0，而之后bit的数量已经小于K，则说明没有办法将整个数列翻转为全1数列。基于这个思路，我们就首先有了一种brutal force的解法，即每次遇到0，就将这一位开始之后的K个比特就进行翻转，完成之后向后移一位继续重复操作。这样我们遍历了整个数列，而在每一位上可能的操作时间复杂度都为O(K)，于是整个解法的时间复杂度就变成了O(KN)，经过测试使用python3会出现超时的情况（而据说C++可以accept）。

而我们换一个角度重新思考一下这个题目，既然每个翻转操作会影响到其后连自身一共K个bit，那么换言之，每个bit亦只会被其之前的K-1个bit所影响，从这个角度来说，我们就可以尝试使用滑动窗的思路来解决这个问题。我们定义一个变量flipped来存储前K-1位上所有的翻转操作带来的最终影响，而用另一个数组isFlipped[N]来记录在位置i处是否进行了翻转，每次滑动之后将flipped和isFlipped[i-K]进行按位异或的操作（和0异或两次与和1异或两次结果都为原始输入）从而抵消当前位置前第K个bit（即刚刚出滑动窗的bit）对flipped的影响。如果当前bit和flipped都为0，则需要进行翻转。如果当前bit和flipped都恰好为1，则说明之前的操作会将当前bit从1翻转为0，我们还是需要对当前bit进行翻转操作。
